Чек-лист для get_triangle_type(a, b, c)

Позитивные тестовые данные

1. Разносторонний треугольник
   Входные данные: 3, 4, 5
   Ожидаемый результат: "nonequilateral"
   Примечание: классический корректный треугольник, у которого все стороны разные.

2. Равнобедренный треугольник
   Входные данные: 3, 3, 2
   Ожидаемый результат: "isosceles"
   Примечание: ровно две стороны равны.

3. Равносторонний треугольник
   Входные данные: 5, 5, 5
   Ожидаемый результат: "equilateral"
   Примечание: все стороны равны.

4. Корректный треугольник со значениями float
   Входные данные: 2.5, 2.5, 3
   Ожидаемый результат: "isosceles"
   Примечание: проверка поддержки конечных вещественных чисел.

Негативные тестовые данные

5. Нулевая длина стороны
   Входные данные: 0, 2, 2
   Ожидаемый результат: IncorrectTriangleSides
   Примечание: сторона треугольника должна быть положительной.

6. Отрицательная длина стороны
   Входные данные: -1, 2, 2
   Ожидаемый результат: IncorrectTriangleSides
   Примечание: отрицательные значения недопустимы.

7. Вырожденный треугольник
   Входные данные: 1, 2, 3
   Ожидаемый результат: IncorrectTriangleSides
   Примечание: не выполняется строгое неравенство треугольника.

8. Невозможный треугольник
   Входные данные: 1, 1, 3
   Ожидаемый результат: IncorrectTriangleSides
   Примечание: одна сторона длиннее суммы двух других.

9. Нечисловой ввод
   Входные данные: "3", 4, 5
   Ожидаемый результат: IncorrectTriangleSides
   Примечание: стороны должны быть конечными вещественными числами.
