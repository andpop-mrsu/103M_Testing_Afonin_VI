# Набор тест-кейсов: нефункциональное тестирование

Набор содержит тест-кейсы для проверки производительности, совместимости, доступности, надёжности и приватности.

## TC-NFR-01
| Поле | Значение |
| --- | --- |
| **Название** | Проверка времени первичной загрузки на настольном устройстве |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Настольное устройство, браузер из поддерживаемого списка, скорость соединения не ниже 10 Мбит/с. |
| **Шаги** | 1. Очистить кэш браузера.<br>2. Открыть приложение.<br>3. Измерить время от начала загрузки до полной готовности интерфейса. |
| **Ожидаемый результат** | Время первичной загрузки не превышает 2 секунд. |

## TC-NFR-02
| Поле | Значение |
| --- | --- |
| **Название** | Проверка времени генерации новой партии на уровне «Профессионал» |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Приложение открыто, выбран уровень «Профессионал». |
| **Шаги** | 1. Нажать кнопку «Новая игра».<br>2. Измерить время до полного отображения нового поля. |
| **Ожидаемый результат** | Новое поле создаётся и отображается не более чем за 200 мс. |

## TC-NFR-03
| Поле | Значение |
| --- | --- |
| **Название** | Проверка визуального отклика на открытие клетки |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Идёт активная партия, доступна скрытая клетка. |
| **Шаги** | 1. Выполнить клик по скрытой клетке.<br>2. Измерить время до визуального изменения состояния клетки. |
| **Ожидаемый результат** | Интерфейс реагирует на действие пользователя не более чем за 100 мс. |

## TC-NFR-04
| Поле | Значение |
| --- | --- |
| **Название** | Проверка корректного отображения интерфейса на экране шириной 360 px |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Открыто приложение в эмуляции или на устройстве с шириной экрана 360 px. |
| **Шаги** | 1. Открыть приложение.<br>2. Проверить расположение основных элементов интерфейса.<br>3. Проверить наличие горизонтальной прокрутки страницы. |
| **Ожидаемый результат** | Основные элементы интерфейса доступны без масштабирования, горизонтальная прокрутка отсутствует. |

## TC-NFR-05
| Поле | Значение |
| --- | --- |
| **Название** | Проверка клавиатурной доступности интерактивных элементов |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Приложение открыто на настольном устройстве. |
| **Шаги** | 1. Перемещаться по интерфейсу с помощью клавиши `Tab`.<br>2. Проверить доступность кнопки «Новая игра», переключателей сложности и кнопки сброса статистики.<br>3. Активировать элементы с клавиатуры. |
| **Ожидаемый результат** | Все обязательные интерактивные элементы получают фокус, имеют заметный индикатор фокуса и активируются с клавиатуры. |

## TC-NFR-06
| Поле | Значение |
| --- | --- |
| **Название** | Проверка устойчивости приложения при временной потере сети |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Приложение полностью загружено, начата партия. |
| **Шаги** | 1. Отключить сетевое соединение после полной загрузки страницы.<br>2. Продолжить выполнение ходов в текущей партии.<br>3. Проверить интерфейс на наличие ошибок. |
| **Ожидаемый результат** | Текущая партия продолжается без критических ошибок, интерфейс не зависает и не перезагружается самопроизвольно. |

## TC-NFR-07
| Поле | Значение |
| --- | --- |
| **Название** | Проверка объёма данных, сохраняемых в localStorage |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | В приложении накоплена статистика, доступ к `localStorage` разрешён. |
| **Шаги** | 1. Открыть инструменты разработчика браузера.<br>2. Найти запись приложения в `localStorage`.<br>3. Оценить общий объём сохраняемых данных. |
| **Ожидаемый результат** | Объём постоянных данных приложения в `localStorage` не превышает 10 КБ. |

## TC-NFR-08
| Поле | Значение |
| --- | --- |
| **Название** | Проверка отсутствия передачи пользовательских данных на внешние серверы |
| **Тип тестирования** | Нефункциональное |
| **Предусловия** | Приложение открыто, инструменты разработчика позволяют анализировать сетевые запросы. |
| **Шаги** | 1. Начать запись сетевой активности.<br>2. Выполнить запуск приложения, несколько игровых действий и сброс статистики.<br>3. Проанализировать исходящие запросы. |
| **Ожидаемый результат** | Статистика и другие пользовательские данные не отправляются на внешние серверы. |
