# Набор тест-кейсов: позитивные сценарии функционального тестирования

Набор содержит тест-кейсы для проверки штатной работы основных функций веб-игры «Сапёр».

## TC-POS-01
| Поле | Значение |
| --- | --- |
| **Название** | Отображение обязательных элементов интерфейса при запуске приложения |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Приложение доступно для открытия в браузере. |
| **Шаги** | 1. Открыть приложение.<br>2. Дождаться завершения первичной загрузки. |
| **Ожидаемый результат** | На экране отображаются игровое поле, кнопка «Новая игра», переключатель уровня сложности, таймер, счётчик мин, блок статистики и индикатор состояния партии. |

## TC-POS-02
| Поле | Значение |
| --- | --- |
| **Название** | Выбор уровня сложности «Новичок» по умолчанию |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Приложение открыто впервые или состояние приложения сброшено. |
| **Шаги** | 1. Открыть приложение.<br>2. Проверить активный уровень сложности без выполнения хода. |
| **Ожидаемый результат** | По умолчанию выбран уровень «Новичок», отображается поле 9 x 9, счётчик мин показывает 10, статус партии — «не начато». |

## TC-POS-03
| Поле | Значение |
| --- | --- |
| **Название** | Создание новой партии кнопкой «Новая игра» |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Приложение открыто, выполнен хотя бы один ход в текущей партии. |
| **Шаги** | 1. Нажать кнопку «Новая игра».<br>2. Проверить состояние поля, таймера и статуса партии. |
| **Ожидаемый результат** | Создано новое поле для текущего уровня сложности, все клетки скрыты, таймер сброшен в 0, статус партии — «не начато». |

## TC-POS-04
| Поле | Значение |
| --- | --- |
| **Название** | Смена уровня сложности на «Любитель» |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Приложение открыто на любом уровне сложности. |
| **Шаги** | 1. Выбрать уровень «Любитель».<br>2. Проверить размеры поля и значения интерфейса. |
| **Ожидаемый результат** | Создано новое поле размером 16 x 16, количество мин равно 40, таймер сброшен в 0, статус партии — «не начато». |

## TC-POS-05
| Поле | Значение |
| --- | --- |
| **Название** | Безопасность первой открытой клетки |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Открыта новая партия, ни одна клетка ещё не открывалась. |
| **Шаги** | 1. Нажать левой кнопкой мыши на любую скрытую клетку.<br>2. Проверить результат открытия клетки. |
| **Ожидаемый результат** | Первая открытая клетка не содержит мину, игра переходит в состояние «в процессе», таймер начинает отсчёт. |

## TC-POS-06
| Поле | Значение |
| --- | --- |
| **Название** | Отображение числа соседних мин в открытой клетке |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Создана новая партия, известна клетка без мины, рядом с которой есть мины. |
| **Шаги** | 1. Открыть целевую клетку.<br>2. Сравнить отображаемое число с фактическим количеством мин в соседних восьми клетках. |
| **Ожидаемый результат** | В клетке отображается корректное число от 1 до 8, соответствующее количеству соседних мин. |

## TC-POS-07
| Поле | Значение |
| --- | --- |
| **Название** | Автоматическое раскрытие связанной области пустых клеток |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Создана новая партия, известна пустая клетка без соседних мин. |
| **Шаги** | 1. Открыть пустую клетку.<br>2. Проверить набор автоматически раскрытых клеток. |
| **Ожидаемый результат** | Автоматически открывается связанная область пустых клеток и граничные клетки с числами; скрытые клетки, не относящиеся к области, остаются закрытыми. |

## TC-POS-08
| Поле | Значение |
| --- | --- |
| **Название** | Установка и снятие флага на настольном устройстве |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Приложение открыто на настольном устройстве, выбранная клетка скрыта. |
| **Шаги** | 1. Нажать правой кнопкой мыши по скрытой клетке.<br>2. Проверить появление флага и изменение счётчика мин.<br>3. Нажать правой кнопкой мыши по той же клетке повторно. |
| **Ожидаемый результат** | После первого нажатия на клетке устанавливается флаг и счётчик мин уменьшается на 1; после повторного нажатия флаг снимается и счётчик мин увеличивается на 1. |

## TC-POS-09
| Поле | Значение |
| --- | --- |
| **Название** | Победа при открытии всех безопасных клеток |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Идёт партия, на поле осталось открыть только безопасные клетки. |
| **Шаги** | 1. Последовательно открыть все оставшиеся безопасные клетки.<br>2. Проверить статус партии, сообщение и статистику. |
| **Ожидаемый результат** | Партия завершается победой, отображается сообщение о победе, поле блокируется, таймер останавливается, счётчики «Всего игр» и «Победы» увеличиваются на 1. |

## TC-POS-10
| Поле | Значение |
| --- | --- |
| **Название** | Сохранение статистики после перезагрузки страницы |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | Завершена хотя бы одна партия, статистика содержит ненулевые значения. |
| **Шаги** | 1. Зафиксировать значения статистики.<br>2. Перезагрузить страницу.<br>3. Проверить блок статистики после загрузки. |
| **Ожидаемый результат** | Значения общего количества игр, побед и поражений сохранены и совпадают с зафиксированными до перезагрузки. |

## TC-POS-11
| Поле | Значение |
| --- | --- |
| **Название** | Подтверждённый сброс статистики |
| **Тип тестирования** | Функциональное (позитивное) |
| **Предусловия** | В статистике есть ненулевые значения, кнопка сброса статистики доступна. |
| **Шаги** | 1. Нажать кнопку сброса статистики.<br>2. В диалоге подтверждения выбрать подтверждение операции.<br>3. Проверить значения в блоке статистики. |
| **Ожидаемый результат** | После подтверждения все показатели статистики устанавливаются в 0 и остаются доступны для дальнейшего накопления. |
