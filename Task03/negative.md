# Набор тест-кейсов: негативные сценарии функционального тестирования

Набор содержит тест-кейсы для проверки устойчивости приложения к некорректным и пограничным действиям пользователя.

## TC-NEG-01
| Поле | Значение |
| --- | --- |
| **Название** | Повторный клик по уже открытой клетке |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | В партии есть хотя бы одна уже открытая клетка. |
| **Шаги** | 1. Нажать левой кнопкой мыши по уже открытой клетке.<br>2. Проверить состояние поля и наличие ошибок. |
| **Ожидаемый результат** | Состояние поля не изменяется, новые клетки не открываются, ошибка интерфейса не возникает. |

## TC-NEG-02
| Поле | Значение |
| --- | --- |
| **Название** | Попытка открыть клетку с установленным флагом |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | На скрытой клетке установлен флаг. |
| **Шаги** | 1. Нажать левой кнопкой мыши по клетке с флагом.<br>2. Проверить состояние клетки и счётчика мин. |
| **Ожидаемый результат** | Клетка не открывается, флаг сохраняется, счётчик мин не изменяется. |

## TC-NEG-03
| Поле | Значение |
| --- | --- |
| **Название** | Попытка установить флаг на уже открытую клетку |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | На поле есть открытая клетка без флага. |
| **Шаги** | 1. Нажать правой кнопкой мыши по открытой клетке.<br>2. Проверить состояние клетки и поля. |
| **Ожидаемый результат** | Флаг не устанавливается, состояние открытой клетки не меняется, другие клетки не затрагиваются. |

## TC-NEG-04
| Поле | Значение |
| --- | --- |
| **Название** | Превышение допустимого количества флагов |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | На поле уже установлено максимальное количество флагов, равное количеству мин выбранного уровня. |
| **Шаги** | 1. Попытаться установить ещё один флаг на скрытую клетку.<br>2. Проверить состояние клетки и счётчика мин. |
| **Ожидаемый результат** | Дополнительный флаг не устанавливается, счётчик мин не уходит ниже допустимого значения, приложение остаётся работоспособным. |

## TC-NEG-05
| Поле | Значение |
| --- | --- |
| **Название** | Быстрые повторные клики по одной скрытой клетке |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | Есть скрытая клетка, по которой ещё не выполнялись действия. |
| **Шаги** | 1. Несколько раз быстро нажать левой кнопкой мыши по одной скрытой клетке.<br>2. Проверить состояние клетки и соседних элементов интерфейса. |
| **Ожидаемый результат** | Клетка обрабатывается корректно один раз, дублированных изменений состояния не возникает, интерфейс не зависает. |

## TC-NEG-06
| Поле | Значение |
| --- | --- |
| **Название** | Перезапуск незавершённой партии кнопкой «Новая игра» |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | Партия находится в состоянии «в процессе», статистика зафиксирована. |
| **Шаги** | 1. Нажать кнопку «Новая игра».<br>2. Проверить статистику после создания новой партии. |
| **Ожидаемый результат** | Новая партия создаётся корректно, текущая незавершённая игра не учитывается как завершённая, статистика не изменяется. |

## TC-NEG-07
| Поле | Значение |
| --- | --- |
| **Название** | Смена уровня сложности в незавершённой партии |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | Партия находится в состоянии «в процессе», статистика содержит исходные значения. |
| **Шаги** | 1. Переключить уровень сложности на другой.<br>2. Проверить параметры нового поля и статистику. |
| **Ожидаемый результат** | Создаётся новое поле выбранного уровня, таймер сбрасывается, незавершённая партия не учитывается как поражение и не влияет на статистику. |

## TC-NEG-08
| Поле | Значение |
| --- | --- |
| **Название** | Действия по полю после завершения партии поражением |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | Партия завершена поражением. |
| **Шаги** | 1. Нажать левой кнопкой мыши по любой клетке.<br>2. Нажать правой кнопкой мыши по любой скрытой клетке.<br>3. Проверить состояние поля. |
| **Ожидаемый результат** | После поражения состояние клеток не меняется, новые флаги не устанавливаются, статистика повторно не увеличивается. |

## TC-NEG-09
| Поле | Значение |
| --- | --- |
| **Название** | Отмена сброса статистики |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | В статистике есть ненулевые значения. |
| **Шаги** | 1. Нажать кнопку сброса статистики.<br>2. В диалоге подтверждения выбрать отмену.<br>3. Проверить блок статистики. |
| **Ожидаемый результат** | Значения статистики остаются без изменений, приложение продолжает работу в штатном режиме. |

## TC-NEG-10
| Поле | Значение |
| --- | --- |
| **Название** | Запуск приложения при недоступном localStorage |
| **Тип тестирования** | Функциональное (негативное) |
| **Предусловия** | В браузере искусственно заблокирован доступ к `localStorage`. |
| **Шаги** | 1. Открыть приложение.<br>2. Проверить доступность игрового процесса и наличие уведомления.<br>3. Выполнить несколько ходов. |
| **Ожидаемый результат** | Приложение запускается без аварийного завершения, игровой процесс доступен, отображается ненавязчивое сообщение о невозможности сохранения статистики. |
